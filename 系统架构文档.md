# 知识库管理系统 — 系统架构文档

---

## 1. 系统概述

本系统是一套基于 **AI 的企业知识库管理平台**，支持文档上传、自动解析分片、向量化存储与 RAG（检索增强生成）问答。  
系统采用前后端分离架构，前端为 Vue 3 SPA，后端为 Spring Boot 3 微模块工程。

---

## 2. 技术栈总览

| 层次 | 技术选型 |
|------|---------|
| **前端框架** | Vue 3.5 + Vite 7 |
| **前端 UI** | Ant Design Vue 4.2 |
| **前端状态管理** | Pinia 3（含持久化） |
| **前端路由** | Vue Router 4 |
| **前端 HTTP** | Axios 1.11 |
| **后端框架** | Spring Boot 3.5.4 / Java 17 |
| **AI 框架** | Spring AI 1.0.1 |
| **LLM / Embedding** | 腾讯混元 Hunyuan-Turbos（Chat）+ Hunyuan-Embedding（1024维） |
| **向量数据库** | PostgreSQL + PgVector（HNSW 索引，余弦距离） |
| **关系数据库** | MySQL 9 |
| **缓存** | Redis（Lettuce） |
| **文档解析** | Apache Tika（TikaDocumentReader） |
| **文本分片** | 自研 ChineseTokenTextSplitter（CL100K_BASE，默认 800 token/块） |
| **ORM** | MyBatis |
| **认证** | JWT + Redis 双重校验 |

## 6. API 设计

### 6.1 统一入口设计

后端采用**单端点 + Header Action** 模式，减少路由分散：

```
POST /api/process
Headers:
  Authorization: Bearer {token}
  ryid: {用户ID}
  action: {操作名称}
Body: JSON
```

| action 值 | 功能 |
|-----------|------|
| createKnowledgeBase | 新建知识库 |
| loadKnowledgeBaseByPage | 分页查询知识库 |
| loadFilesByKnowledgeBase | 查询知识库下的文件列表 |
| deleteFile | 删除文件（软删除） |
| parseFile | 触发文件异步解析 |
| toggleFileEnabled | 切换文件是否参与 RAG 检索 |

### 6.2 文件上传接口

```
POST /api/process/file?knowledge_base_id={id}
Content-Type: multipart/form-data
Body: file（文件二进制）
```

### 6.3 统一响应格式

```json
{
  "code": 0,
  "msg": "success",
  "data": { ... }
}
```

### 6.4 前端 HTTP 拦截器

`src/utils/request.js` 中通过 Axios 拦截器：
- **请求拦截**：自动注入 `Authorization: Bearer {token}` 和 `ryid` Header
- **响应拦截**：`code === 0` 正常返回 data，否则弹出错误提示

## 7. 认证与授权

### 7.1 登录流程

```
POST /api/auth/login { username, password }
    ↓
查询 MySQL: sys_user WHERE username=? AND password=?
    ↓
生成 JWT Token（JwtUtil）
    ↓
写入 Redis: km:logins:{ryid} = {token, username, ryid}（TTL 10小时）
    ↓
返回: { token, uuid, username, ryid }
```

### 7.2 请求鉴权流程

```
HTTP 请求
    ↓
TokenAuthenticationFilter
    ├── 白名单跳过（/api/auth/**, /swagger-ui/**, /v3/api-docs/**）
    ├── 解析 Header: Authorization + ryid
    ├── 验证 JWT 签名
    ├── 对比 Redis 中缓存的 Token
    ├── 刷新 Redis TTL
    └── 写入 UserContext（ThreadLocal）→ 业务层直接取用户信息
```

### 7.3 前端路由守卫

`router/index.js` 中通过 `beforeEach`：
- 未登录访问受保护路由 → 跳转 `/login`
- 已登录访问登录页 → 跳转 `/app`

## 8. 文档解析与向量化流程

```
用户点击"解析"按钮
    ↓
前端 POST /api/process (action=parseFile, { id: fileId })
    ↓
Controller → KnowledgeService.parseFile()
    ↓
DocumentParseService.parseAsync(fileId)  ← @Async 异步执行
    │
    ├─ 1. 更新 DB: parsing_status = 'processing'
    │
    ├─ 2. 读取文件
    │      TikaDocumentReader（支持 PDF/Word/Excel/PPT/TXT/HTML 等 13+ 格式）
    │
    ├─ 3. 文本分片
    │      ChineseTokenTextSplitter
    │      - 编码器：CL100K_BASE（cl100k_base tiktoken）
    │      - 默认 chunk size：800 tokens
    │      - 最小 chunk 字符数：350
    │      - 在中文标点（。？！）处断句
    │
    ├─ 4. 注入元数据
    │      每个分片附加: { file_id, knowledge_base_id, file_name }
    │
    ├─ 5. 批量 Embedding
    │      EmbeddingModel.embed(texts)
    │      → 腾讯混元 Hunyuan-Embedding API
    │      → 输出 1024 维 float[] 向量
    │
    ├─ 6. 写入向量库
    │      PgVectorStore.add(chunks)
    │      → PostgreSQL vector_store 表（HNSW 索引）
    │
    └─ 7. 更新 DB: parsing_status = 'done', chunk_count = N
           （失败时置为 'error'）
```

## 9. 前端状态管理

### Pinia Store

| Store | 持久化 | 内容 |
|-------|--------|------|
| `useBackendsStore` | 否 | 后端 baseUrl |
| `useCurrentKBStore` | 是 | 当前选中的知识库 `{ id, name }` |
| `useAuthStore` | 是 | 登录用户信息 `{ token, username, ryid }` |

### 页面导航与知识库上下文

用户点击知识库卡片时，`KnowledgeManager2.vue` / `HomePage.vue` 调用：
```javascript
kbStore.setCurrentKB({ id: item.id, name: item.name })
router.push('/knowledgedetail')
```
子页面 `FileManager.vue` 从 `currentKBStore` 读取 `kbId`，作为所有文件接口的参数。
